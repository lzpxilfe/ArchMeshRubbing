# ArchMeshRubbing 개발철학

이 문서는 ArchMeshRubbing 프로젝트의 개발 방향과 의사결정 기준을 정의한다.

0. 엄밀성
고고학 자료는 실측 가능해야 하며, 측정 가능한 데이터를 다루므로 엄밀함이 필수다.

1. 직관성
사용설명서를 보지 않아도 대략적으로 사용 흐름을 파악할 수 있어야 한다.

2. 개방성
오픈소스로 제공하며 사람들에게 유익함을 제공한다. 지식은 전유물이 아니라는 생각에 입각해 공개된 연구자료를 레퍼런스로 명확히 언급한다.

3. 경쟁성
고고학 업계의 기존 강자 프로그램과 경쟁해 승리할 수 있는 품질을 만든다. 목표는 더 많은 사람에게 실질적인 이로움을 주는 것이다.

4. 쾌활성
무거운 데이터를 다루더라도 쾌적한 구동 환경과 작업 경험을 제공한다.

5. 범용성
초기 목표였던 물리적 기와 탁본 데이터 제작을 넘어 3D 데이터 자체를 폭넓게 다루는 프로그램으로 발전한다. 처리된 데이터는 Illustrator 후작업에 적합해야 하며, 장기적으로 국제적인 프로그램으로 확장한다.

핵심 문장
정확한 고고학 3D 데이터를 누구나 쉽게, 빠르게, 개방적으로 다루게 한다.
